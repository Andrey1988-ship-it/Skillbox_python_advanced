# Microblog API Project

Это полнофункциональное веб-приложение (клон микроблога), построенное на современном стеке Python. Проект включает в себя асинхронный бэкенд, готовую базу данных и интегрированный фронтенд.

## 🚀 Стек технологий

*   **Backend:** FastAPI (Python 3.10)
*   **Database:** PostgreSQL
*   **ORM:** SQLAlchemy 2.0 (Async)
*   **Frontend:** Vue.js (интегрирован в папку `dist`)
*   **Containerization:** Docker & Docker Compose
*   **Testing:** Pytest (Asyncio)
*   **Linting:** Flake8 & Black

## 🛠 Установка и запуск

Для запуска проекта вам понадобится только [Docker](https://docker.com).

1. **Клонируйте репозиторий:**
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd microblog
   ```

2. **Запустите проект:**
   ```bash
   docker-compose up --build
   ```

Приложение будет доступно по адресу: [http://localhost:8000](http://localhost:8000)
Документация API (Swagger): [http://localhost:8000/docs](http://localhost:8000/docs)

## 🧪 Тестирование

Тесты запускаются внутри Docker-контейнера. Мы используем отдельную тестовую базу данных для чистоты проверок.

```bash
docker exec -it microblog-backend-1 env PYTHONPATH=. pytest -c pytest.ini tests/test_api.py
```

## 🧹 Линтинг и качество кода

Проект настроен на соблюдение стандартов PEP8. Проверка линтером проходит автоматически через **GitHub Actions** при каждом push-запросе.

Локальный запуск линтера:
```bash
docker exec -it microblog-backend-1 flake8 .
```

## 📋 Реализованный функционал

- **Твиты:** Создание, получение ленты, удаление (с проверкой прав доступа).
- **Медиа:** Загрузка изображений и привязка их к твитам.
- **Пользователи:** Просмотр своего профиля и чужих профилей.
- **Социальные связи:** Система подписок (фолловинг) и лайков.
- **SPA:** Раздача фронтенда напрямую через FastAPI сервер.

## 📦 Структура проекта

- `app/api/` — маршруты (endpoints) для твитов и пользователей.
- `app/core/` — конфигурация базы данных.
- `app/models/` — модели SQLAlchemy (Base, User, Tweet, Media).
- `dist/` — собранные статические файлы фронтенда.
- `tests/` — асинхронные тесты функционала.
